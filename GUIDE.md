## 🔍 SEO (Search Engine Optimization) — The Big Win

This is where the real value is. We didn't just build a pretty website — we built a **Google ranking machine**.

### What We Created for SEO

| Page Type | Count | Example URL |
|-----------|-------|-------------|
| **Homepage** | 1 | `/` |
| **Category Pages** | 3 | `/services/residential`, `/services/commercial`, `/services/automotive` |
| **Service Pages** | 29 | `/services/residential/smart-lock-installation` |
| **City Pages** | 24 | `/service-areas/cleveland`, `/service-areas/lakewood` |
| **Service Areas Index** | 1 | `/service-areas` |
| **Total** | **58 unique pages** | All optimized for Google |

### What Each Page Includes (for Google)
- ✅ **Unique title tag** — Under 60 characters, includes main keyword
- ✅ **Meta description** — Under 160 characters, compelling and keyword-rich
- ✅ **Canonical URL** — Tells Google which is the "real" version of each page
- ✅ **JSON-LD structured data** — Machine-readable info that helps Google show rich results (star ratings, business hours, service areas, pricing)
- ✅ **Breadcrumb navigation** — Home → Category → Service (Google loves this)
- ✅ **Semantic HTML** — Proper `<h1>`, `<h2>`, `<main>`, `<section>` tags
- ✅ **Internal linking** — Pages link to each other, spreading SEO value

### Types of Structured Data (JSON-LD)
- **LocalBusiness** — Business name, phone, address, hours, ratings, service areas
- **FAQPage** — Questions & answers shown directly in Google results
- **Organization** — Team members with roles and certifications
- **Service** — Individual service with pricing and area served
- **OfferCatalog** — Full catalog of all 29 services

### Why This Matters
When someone in Lakewood searches "locksmith near me" or "car lockout Parma," Google now has **dedicated pages** to show them — not just one generic homepage. This is how local businesses dominate Google Maps and search results.

**Typical admin tasks:**
- To add a new city: Go to **Service Areas → Create New** → enter city name, coordinates, response time → **Save**. The map updates, a new city landing page auto-generates, and the sitemap updates.
- To update pricing: Go to **Services → [Service Name]** → change the price → **Save**. All pages showing that price update automatically.
- To add a team member: Go to **Team Members → Create New** → fill in bio, upload photo, add certifications → **Save**. They appear on the website and in Schema.org data.
- To approve a review: Go to **Reviews** → find the pending review → check "Approved" → **Save**. It appears on the website and the star rating recalculates.

### Technical Detail: How It Works Under the Hood

The current website uses **static TypeScript data files** (`services.ts`, `locations.ts`, `team.ts`, etc.) that map 1:1 to Payload CMS collections. When the backend is deployed:

1. Each `import { data } from '@/data/file'` becomes a `fetch()` to Payload's REST API
2. The React components **don't change at all** — only the data source swaps
3. Pages use **ISR (Incremental Static Regeneration)** — they're pre-built for speed but refresh automatically when content changes in the CMS
4. All media (images, PDFs, certificates) is served through a CDN for fast loading worldwide
